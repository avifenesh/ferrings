# Code of Conduct

ferrings is a technical project, but the work is still done by people. Keep
discussion direct, specific, and respectful.

## Expected Behavior

- Be precise about bugs, tradeoffs, and evidence.
- Assume good intent while still asking for proof.
- Keep criticism focused on code, docs, benchmarks, and release behavior.
- Make room for different experience levels in issues and pull requests.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information without permission.
- Sustained off-topic disruption of issues, pull requests, or discussions.
- Misrepresenting benchmark results, security impact, or release state.

## Reporting

For security issues, follow `SECURITY.md`.

For conduct issues, open a private security advisory or contact the maintainer
through the repository owner profile. Reports should include links, timestamps,
and a short description of what happened.

Maintainers may remove comments, close threads, block accounts, or take other
reasonable action to keep the project workable.
